## Security Controls Verified

### 1. SQL Injection Prevention
- All queries use parameterized statements (`sqlite3_bind_text`)
- No string concatenation in SQL construction
- Input validation on all user parameters

### 2. Password Storage
- PBKDF2-SHA256 with 10,000 iterations
- Unique cryptographically random salt per account
- Salt and hash stored as binary BLOBs

### 3. Token Management
- UUID-based refresh tokens (cryptographically random)
- Refresh tokens stored in dedicated `refresh_tokens` table
- Automatic token cleanup on account deletion
- Session management via `ServiceDatabases`

### 4. Input Validation
- CBOR type tag validation in `CBOR.m`
- HTTP header size limits enforced
- Rate limiting on API requests
- NSID format validation for ATProto methods

### 5. Cryptography
- ECDSA signing via secp256k1
- JWT token handling
- DPoP proof generation support
- Secure key storage in Keychain
