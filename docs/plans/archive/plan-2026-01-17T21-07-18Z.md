### 1) WebAuthn Base64URL handling and input validation (medium)
- Location: `ATProtoPDS/Sources/Auth/WebAuthnVerifier.m:14` `ATProtoPDS/Sources/Auth/WebAuthnVerifier.m:55` `ATProtoPDS/Sources/Auth/WebAuthnVerifier.m:154`
- Risk: WebAuthn uses Base64URL; direct Base64 decoding can fail for valid payloads. Also untrusted JSON/CBOR types are assumed to be dictionaries, which can crash or mis-handle malformed inputs.
- Suggested edits:
  - Normalize Base64URL before decoding `clientDataJSON`, `attestationObject`, `authenticatorData`, and `signature`.
  - Validate JSON types (dictionary check) and CBOR decode result before dereferencing.

Proposed changes (sketch):
```objc
// Add helper: normalize Base64URL to Base64 with padding.
+ (NSString *)normalizeBase64URL:(NSString *)base64url {
    if (!base64url) return nil;
    NSString *base64 = [[base64url stringByReplacingOccurrencesOfString:@"-" withString:@"+"]
                                  stringByReplacingOccurrencesOfString:@"_" withString:@"/"];
    while (base64.length % 4 != 0) { base64 = [base64 stringByAppendingString:@"="]; }
    return base64;
}

// Use for all incoming fields:
NSData *clientDataJSON = [[NSData alloc] initWithBase64EncodedString:[self normalizeBase64URL:clientDataJSONB64] options:0];
// ... same for attestationObject/authenticatorData/signature

// Validate JSON/CBOR types before subscripting
id clientObj = [NSJSONSerialization JSONObjectWithData:clientDataJSON options:0 error:nil];
if (![clientObj isKindOfClass:[NSDictionary class]]) { /* return error */ }
CBORValue *attestationParams = [CBORValue decode:attestationObject];
if (!attestationParams || attestationParams.type != CBORTypeMap) { /* return error */ }
```

### 2) SecKeyRef leak when generating signing key (low)
- Location: `ATProtoPDS/Sources/Database/ActorStore/ActorStore.m:971` `ATProtoPDS/Sources/Database/ActorStore/ActorStore.m:1020`
- Risk: `SecKeyCreateRandomKey` returns retained `SecKeyRef`. `_setSigningKeyUnsafe` retains again, and `privateKey` is not released on success, leaking the key.
- Suggested edit:
  - Release `privateKey` after `_setSigningKeyUnsafe`.

Proposed change (sketch):

_setSigningKeyUnsafe(&_signingKey, privateKey);
CFRelease(privateKey);
