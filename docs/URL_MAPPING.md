# URL Mapping: Jekyll to VitePress

This document tracks URL changes from the Jekyll documentation to the VitePress documentation.

## Migration Notes

The VitePress migration maintains the same file structure and naming convention as Jekyll, so most URLs remain unchanged. The primary difference is that VitePress uses extensionless URLs (no `.html` or `.md` extensions in the browser).

## URL Format Changes

### Jekyll Format
```
https://pds.garazyk.xyz/docs/01-getting-started/overview.html
```

### VitePress Format
```
https://pds.garazyk.xyz/docs/01-getting-started/overview
```

## Automatic Redirects

VitePress automatically handles the following:

1. **Extension removal**: `/docs/page.html` → `/docs/page`
2. **Trailing slashes**: `/docs/page/` → `/docs/page`
3. **Index files**: `/docs/section/` → `/docs/section/index`

## Pages That Changed

### None Identified

During the migration, all pages maintained their original paths and filenames. No manual redirects are required at this time.

If pages are renamed or moved in the future, add them to the nginx configuration:

```nginx
# Example redirect
rewrite ^/docs/old-path$ /docs/new-path permanent;
```

## Testing Redirects

To test that old Jekyll URLs redirect properly:

```bash
# Test that .html extension redirects work
curl -I https://pds.garazyk.xyz/docs/01-getting-started/overview.html

# Should return 200 OK (VitePress handles this via try_files)
```

## External References

If you have external links pointing to the old Jekyll documentation, they should continue to work due to:

1. Same base path (`/docs`)
2. Same file structure
3. VitePress's flexible routing (handles `.html` extensions)

## Bookmarks

Users with bookmarked pages should update their bookmarks to remove `.html` extensions, but existing bookmarks will continue to work.

## Search Engine Updates

Search engines will automatically discover the new URL format through:

1. Sitemap.xml (generated by VitePress)
2. Canonical URLs in page metadata
3. Natural crawling and indexing

No manual intervention required for SEO.
