# XRPC Registration ‚Äî NSID Constants Plan

**Status:** Planned, not enacted (plan-only). No production code changed by this document.
**Date:** 2026-07-11
**Lineage:** Architecture review candidate 3 ("Delete XrpcHandler's 119 pass-throughs").
The review proposed *deleting* the shallow layer; grilling inverted the direction to
*generating* a uniform, type-safe registration surface for all methods instead.

---

## Why

`XrpcHandler.h` exposes **124 methods, of which 117 are one-line `register‚Ä¶:` delegators**
(`registerComAtprotoServerCreateAccount:` ‚Üí `[self registerMethod:@"com.atproto.server.createAccount" handler:handler]`).
It is a **shallow** layer: each method hides a single string constant. Worse, it is
*inconsistently* used ‚Äî only **7 of 34 route packs** call the convenience methods
(116 call sites); the other **27 packs already register with raw string literals**
(`registerMethod:@"app.bsky.feed.getLikes"`). So the codebase has two idioms and ~332
registration call sites total, none of which share a single source of truth for method IDs.

The chosen direction: generate one canonical set of **method-ID constants** from the
lexicons, unify all packs on it, delete the 117 convenience methods, and guard against
drift with a lint. This deepens `XrpcHandler` (interface shrinks 124 ‚Üí ~7) *and* gives every
pack the same type-safe, discoverable registration surface.

## Decision trail (from the grilling session)

1. **Direction: generate a type-safe surface for all, don't just delete.** The value of the
   117 methods was never the delegation ‚Äî it was compile-checked, discoverable method IDs.
   Generation preserves that value for *all* methods while removing the hand-maintenance cost
   that made the layer a liability.
2. **Artifact: plain `NSString * const` NSID constants.** Not generated convenience methods
   (would balloon `XrpcHandler` to ~400 ‚Äî a bigger shallow surface) and not a typed enum
   (enum‚Üîstring indirection). Constants keep `XrpcHandler` deep and are reusable data.
3. **Generator home: extend `packages/gruszka/scripts/generate.ts`.** Gruszka already
   enumerates all **331 method NSIDs** and emits `generated_types.ts` (the TS client). Adding
   an ObjC emitter beside it keeps the ObjC NSIDs in lockstep with the client from one
   `deno task generate`.
4. **Committed + CI drift-checked; `GZ` prefix.** The generated `GZXrpcNSID.{h,m}` is checked
   in (like gruszka's own `generated_types.ts`), so the C build never depends on the Deno
   toolchain; CI re-runs the generator and fails on drift. Prefix matches `GZXrpcRouteSupport`
   / `GZLogger`.
5. **Rollout: pilot one pack, then phase** (see below).
6. **Enforcement: plain constants + a `narzedzia` lint ‚Äî NOT a compile-time typed wrapper.**
   Initially a compile-time-typed `GZNSID` wrapper was chosen, then reversed after a cost
   analysis (see "Rejected alternatives"). Plain constants already make a mistyped constant
   *name* a compile error; the lint closes the only remaining gap (a fresh raw literal).

## Key evidence

- `Network/XrpcHandler.h` ‚Äî 124 methods; 117 are `register(Com|App|Chat|Tools)‚Ä¶:` pure
  delegators (verified: none use the `registerMethod:middlewares:handler:` variant).
- 27 of 34 `Network/*Pack.m` register with raw `registerMethod:@"‚Ä¶"` literals; 7 packs
  (`XrpcAdminPack`, `XrpcServerPack`, `XrpcRepoPack`, `XrpcSyncPack`, `XrpcIdentityPack`,
  `XrpcLabelPack`, `XrpcModerationPack`) use the convenience methods (116 call sites).
  ~332 registration call sites total.
- No method-ID constants exist anywhere today; IDs live only in the 117 method bodies and in
  the 27 packs' literals.
- The dispatcher core is irreducibly `NSString`-keyed:
  `methodHandlers` is `NSMutableDictionary<NSString *, XrpcMethodHandler>`
  (`XrpcHandler.m:18`), plus `internalHandlers`, `protectedMethods` (`:96`), and lookup by
  incoming request string (`:252`). Incoming method-ids are arbitrary strings, so the
  read/dispatch/auth path cannot be typed regardless.
- `packages/gruszka/scripts/generate.ts` + `lexicons.ts` enumerate 331 method NSIDs and
  produce `generated_types.ts`.

## Target artifact

Generated `Garazyk/Sources/Network/Generated/GZXrpcNSID.{h,m}` (331 constants):

```objc
// GZXrpcNSID.h  ‚Äî @generated by gruszka from the lexicons; do not edit by hand
extern NSString * const GZNSIDComAtprotoServerCreateAccount;   // com.atproto.server.createAccount
extern NSString * const GZNSIDAppBskyFeedGetLikes;             // app.bsky.feed.getLikes
// ‚Ä¶ 329 more

// GZXrpcNSID.m
NSString * const GZNSIDComAtprotoServerCreateAccount = @"com.atproto.server.createAccount";
NSString * const GZNSIDAppBskyFeedGetLikes           = @"app.bsky.feed.getLikes";
```

**No change to the registration API.** Packs use the existing
`registerMethod:(NSString *)methodId handler:` with a constant instead of a literal ‚Äî a
drop-in replacement for both idioms:

```objc
[dispatcher registerMethod:GZNSIDComAtprotoServerCreateAccount
                   handler:^(HttpRequest *request, HttpResponse *response) { /* ‚Ä¶ */ }];
```

The constants are free compile-time literals (no `GZNSID` object, no `.stringValue`, no
load-time init) and are reusable anywhere an NSID string is needed (dispatch, tests, logging,
the client). A mistyped constant *name* is an "undeclared identifier" compile error.

## Rollout (pilot-first, phased)

1. **Land the generator + committed `GZXrpcNSID.{h,m}`.** Additive; no pack changes, nothing
   breaks. Add a gruszka generator-output test (mirrors `generate_test.ts`).
2. **Pilot `XrpcServerPack`** ‚Äî convert its convenience-method calls to
   `registerMethod:GZNSID‚Ä¶ handler:`. Prove the generated names + the pattern end-to-end.
3. **Convert the remaining 6 convenience packs; delete the 117 convenience methods.**
   `XrpcHandler` interface shrinks 124 ‚Üí ~7. This is the interface-deepening win.
4. **Convert the 27 raw-literal packs** (`@"‚Ä¶"` ‚Üí `GZNSID‚Ä¶`) ‚Äî the uniformity win. Mechanical;
   can land pack-by-pack.
5. **Enforcement capstone: enable the `narzedzia` lint** forbidding `registerMethod:@"literal"`
   (must use a `GZNSID‚Ä¶` constant). Enabled only after phase 4, else it trips on unmigrated
   packs. Fits the existing boundary-check tooling.

**Deletion-test payoff:** the 117 pass-throughs vanish (interface 124 ‚Üí ~7); their content
(the method-ID strings) is concentrated into one generated source of truth instead of being
duplicated across method bodies and 27 packs' literals.

## Verification

- Routing behavior is unchanged ‚Äî the constants resolve to the same strings the packs used
  before; the dispatcher is untouched. Existing `Tests/Network` / `Tests/XRPC` dispatch tests
  are the regression net.
- New: a gruszka generator test asserting the emitted `GZXrpcNSID.{h,m}` matches the committed
  output (drift guard), and a CI step running `deno task generate` + `git diff --exit-code`.
- The `narzedzia` lint gets its own unit test (flags a raw literal, passes a constant).

## Rejected alternatives (with reasons)

- **Delete-only (the review's original proposal).** Rejected: the method-ID strings are worth
  a single typed source of truth; deleting down to raw literals leaves 34 packs stringly-typed
  with no shared constants.
- **Generate convenience methods for all (~400).** Rejected: re-creates the shallow surface at
  3√ó the size ‚Äî a 400-method generated `XrpcHandler` interface a reader/AI scrolls past.
- **Typed method-ID enum.** Rejected: the enum value isn't the NSID string, so every
  dispatch/log/test needs an `‚Ä¶ToString()` hop.
- **Compile-time-typed `GZNSID` wrapper class.** Rejected after cost analysis, and worth
  recording so it isn't re-proposed:
  - In ObjC (not Swift), `GZNSID * const X = [GZNSID nsidWithString:@"‚Ä¶"]` is **not a constant
    expression** ‚Äî it won't compile as a `const` global. The 331 entries would become
    load-time-initialized mutable globals or 331 accessor functions (the big surface returns).
  - Safe use as dictionary keys / with `==` needs interning or `isEqual:`/`hash` + a "never
    `==`" rule; `NSString` gives this for free and is already the dispatcher's key type.
  - The dispatcher core is irreducibly `NSString`-keyed and incoming ids are arbitrary
    strings, so the wrapper covers only the **~332 registration write sites (~2%)** and is
    unwrapped to `.stringValue` immediately inside registration.
  - Net: the wrapper pays those costs to move **one** drift case (a fresh raw literal) from a
    CI failure to a compile failure ‚Äî which the `narzedzia` lint already catches. Plain
    constants + lint capture ~95% of the value at ~5% of the cost.

  *(This rejection has a load-bearing, ObjC-specific reason a future reviewer would otherwise
  re-raise ‚Äî a candidate for an ADR.)*
