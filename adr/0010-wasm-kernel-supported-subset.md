# ADR 0010: WASM Kernel Supported Subset After the Capability Baseline

## Status

Accepted — 2026-07-23 (operator-confirmed; workstream 05 E1, mega-plan Phase 5 item 1)

## Context

The phase-10 capability baseline
(`objc-jupyter-wasm/scripts/run-capability-baseline.sh`, run 2026-07-22)
made the kernel's support boundary test-generated: 91/91 runtime gap
probes, 18/18 compatibility-corpus cases, 22 demo notebooks green (the 14
skipped cells are intentional exercise placeholders, not gaps), and a
passing Chromium worker smoke. The matrix
(`objc-jupyter-wasm/docs/capability-matrix.md`) supersedes every
hand-maintained status table.

Against that baseline, the interpreter's only remaining *language* gaps
are the four probes the gap suite skips by commented-out cases:

1. the `->` member-access operator sends the parser into an infinite
   loop — a robustness bug, since any notebook cell containing
   `ptr->field` hangs the browser kernel rather than erroring;
2. C function definitions are unsupported, even though the tutorial
   snippet policy invites snippets that "mirror production structure"
   and Garazyk production sources use file-scope C helper functions
   pervasively (e.g. `UIEscaped()` in AdminUIServer);
3. `@encode` fails with a parse error;
4. `@synchronized` fails with a parse error.

Everything else the matrix defers is not language behavior: the
Emscripten compiled-cell plane, compile service, side-module loading,
cross-cell inspection, completion, introspection, and full Jupyter
exception presentation.

The E1 rule for this choice: favor language behavior required by the
notebooks and the Garazyk compatibility corpus over broad Foundation
imitation.

## Decision

The next supported subset is the two notebook-relevant language gaps,
plus a hard robustness invariant; the other two gaps become
intentionally unsupported with diagnostics; the compile plane stays
deferred.

1. **Parser termination invariant (P0 within this lane).** No input may
   hang the parser. The `->` infinite loop is fixed first, and the fix
   carries a regression probe. This invariant outranks any feature:
   a browser kernel that hangs is worse than one that rejects.
2. **Support `->` member access.** Implement object ivar access
   (`self->_ivar`, `obj->ivar`) and struct-pointer member access,
   consistent with the already-supported dot access on structs and
   ivar/KVC machinery. Positive gap probes replace the skip comment.
3. **Support top-level C function definitions** with value-typed
   parameters and returns (the tutorial-snippet shape). No function
   pointers, varargs beyond `NSLog`-style passthrough, or `static`
   linkage semantics are promised. Positive gap probes replace the
   skip comment.
4. **`@encode` and `@synchronized` are intentionally unsupported.**
   Both must parse to a clean, explicit diagnostic (the same fail-safe
   shape the compatibility runner uses for unsupported production
   APIs), never a parse error or hang. `@synchronized` is declared
   unsupported to stay consistent with the snippet policy's exclusion
   of threading primitives; a semantically safe single-threaded no-op
   remains a permissible future upgrade without revisiting this ADR.
5. **The compiled-cell plane and Jupyter UX features remain deferred**
   and are out of scope for interpreter work; nothing here schedules
   them.

Acceptance for the implementation lane: the four skip comments in
`tests/test-runtime-gap-probes.mjs` become active probes (positive for
items 2-3, diagnostic-asserting for item 4), the capability matrix is
regenerated by the baseline runner, and notebooks plus the
compatibility corpus stay green.

## Consequences

- Tutorial snippets can mirror production Objective-C structure
  (helper functions, ivar access) without rewriting around missing
  syntax, which is the corpus's stated purpose.
- The kernel gains a stated termination guarantee, which the browser
  deployment implicitly depends on and no test previously pinned.
- `@encode`/`@synchronized` users get a truthful diagnostic instead of
  a confusing parse error; no Foundation-imitation scope creep occurs.
- The subset is small enough to ship as one interpreter lane with the
  existing baseline runner as its verification gate; rollback is
  reverting the interpreter slice, since the probes and matrix
  regenerate deterministically from whatever the kernel supports.
