# External Guidance

This skill is grounded in the following external guidance:

- Google developer style guide:
  - Avoid excessive claims and jargon.
  - Keep docs timeless (avoid release-tied qualifiers in long-lived product docs).
  - Use API reference comments that are factual and actionable.
- Divio/Diataxis documentation system:
  - Separate tutorials, how-to guides, reference, and explanation by reader intent.
- Apple documentation references:
  - Use structured symbol/header comments for Objective-C APIs.
  - Use explicit parameter/return sections for Quick Help-friendly comments.
- Doxygen command reference:
  - Use `@param` and `@return`/`@result` style tags consistently.
- GitHub skills documentation:
  - Keep skills in dedicated directories with focused `SKILL.md` frontmatter + optional references/scripts.
