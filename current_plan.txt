ObjC Jupyter WASM Kernel — Revised Plan
═══════════════════════════════════════════════════════════════════════════════

Context

This plan supersedes the prior structural-refactor-only plan. It folds in:
  • Three correctness fixes (C1, C3, C4) — IMPLEMENTED, build verified
  • Three uncommitted dirty-baseline fixes (broken comment, debug code)
  • One pre-existing kernel-smoke regression (Counter class redefinition)
  • The 12-step structural refactor from the prior plan
  • Four behavioral phases (perf, errors, inspect, hygiene)

The kernel is a single-file 8,452-line Objective-C tree-walker compiled to
WebAssembly via wasm32-wasi (LLVM 21) with libobjc2 linked in. The build
system is Nix (objc-jupyter-wasm/nix/kernel-wasm.nix); the test surface is
tests/kernel-smoke.mjs run as a flake check, plus the demo notebook harness
at tests/run-notebooks.mjs.

Goals

  1. Land correctness fixes without regressing existing tests
  2. Unblock the kernel-smoke check (currently failing on a pre-existing bug)
  3. Decompose the monolithic interpreter without changing behavior
  4. Add hash-backed lookups, error spans, frame stacks, richer inspect
  5. Maintain a working build at every commit

──── Commit 0B — Correctness fixes C1, C3, C4 (already implemented) ────

C1 — selector-builder buffer overflow (objc_interpreter.c:2438)

  Bug: cstr_copy bounds the byte write but `sel_len += part_len` advances
       by the untruncated length. The next `sel_name[sel_len] = ':'`
       (line 2457) then writes past the 256-byte buffer.

  Fix: before each append, compute (sel_len + part_len + 2) and emit a
       parser_error if it exceeds sizeof(sel_name). Reserves room for the
       trailing colon and null.

C3 — FDObj:overflow magic-string sentinel (lines 2563, 2577)

  Bug: on string-pool exhaustion, parse_message_send returned the literal
       string "FDObj:overflow" cast to id. This pointer is in .rodata
       (not the string pool), so `is_string_pool_pointer` returns false
       and downstream FDObj-aware code partially recognised it as a
       Foundation marker for class "overflow", causing silent class-name
       confusion.

  Fix: replace both sentinels with `parser_error(p, "string pool exhausted
       ([alloc])")` + `return value_void()`.

C4 — silent identifier truncation (13 sites)

  Bug: `cstr_copy(buf, ident, 64)` silently truncates identifiers >63
       chars. Later table lookups compare the full source name against
       the truncated stored copy and miss. Affected: class name, super
       name, property type/name, ivar name, method-arg name, var-decl
       name, for-in var, block arg, dot-syntax property name.

  Fix: new `cstr_copy_checked(dst, src, cap)` returning 1 on truncation,
       and `copy_identifier_or_error(p, dst, src, cap, kind)` that emits
       a structured parser_error naming the kind and offending prefix
       (e.g. "class name too long (max 63): 'AVeryVeryLong...'").

  All 13 user-input identifier sites now use the checked variant. Internal
  copy sites (NSFloat:, FDObj: prefixes; mi->arg_names from already-validated
  arg_names) remain on cstr_copy.

Action: commit the remaining hunks (C1, C3, C4):

  git add objc-jupyter-wasm/kernel/objc_interpreter.c
  git commit -m "fix(objc-jupyter-wasm): repair selector overflow, FDObj sentinel, silent truncation"

──── Commit 0C — Smoke test fix R1 (Counter Phase 5/8) ────

Issue R1: kernel-smoke Phase 8 fails with
          `-[unknown setCount:] does not respond to selector`
          and `prop-count=7` instead of `5`.

Root cause: kernel-smoke.mjs Phase 5 (line 543, `dotGetterTest`) registers
            `Counter : Object` with method `count { return 7; }`. Phase 8
            (line 667, `propTest`) re-declares
            `Counter : NSObject` with `@property count` and `@synthesize`.

            parse_interface (objc_interpreter.c:4290–4308) and
            parse_implementation (objc_interpreter.c:4621–4644) detect
            already-registered classes and skip the entire body up to
            @end. So Phase 8's @property is never recorded and `[c count]`
            still routes to Phase 5's hard-coded `return 7`.

This regression exists in HEAD and in the dirty-baseline. C1/C3/C4 do
not cause it (verified by reverting all three; identical failure).

There are two correct fixes; pick the one that matches what the
interpreter is meant to support:

OPTION A (RECOMMENDED): rename in the test
  Rationale: the interpreter does not support class redefinition.
             Idempotency-as-skip is the documented contract. Tests should
             not silently rely on undefined re-declaration.
  Change: in objc-jupyter-wasm/tests/kernel-smoke.mjs, rename Phase 8's
          `Counter` to e.g. `PropCounter` (lines 668, 673, 680, 681, 682,
          684, 685). Also update the GC-related reuse at line 798–808.
  Effort: ~10-line test edit. Low risk.

OPTION B: relax the idempotency guard to merge declarations
  Rationale: aligns with real ObjC where re-running a cell that adds a
             property to an existing class should work.
  Change: in parse_interface, when `existing` is found, do NOT skip to
          @end. Instead, continue parsing but skip class allocation and
          ivar block. Add @property declarations into g_properties only
          if no entry exists for (class_name, prop_name). Same for
          parse_implementation: process @synthesize and add new methods
          via class_addMethod for selectors not already in g_methods for
          this class. Drop the early returns at lines 4307 and 4643.
  Effort: ~80 LOC interpreter change. Requires a regression notebook
          covering: (a) re-declare class with same property — no dup,
          (b) re-declare class with new property — added, (c) re-declare
          @implementation adding a method — added, (d) re-declare
          method body — last-write-wins or rejected (decide & document).

Pick: Option A first (unblocks CI). File a follow-up issue for Option B
      and ship it in Phase 4 (errors track) or as standalone work — it
      improves notebook UX where users iterate by re-running cells.

STATUS: Option A applied in worktree. tests/kernel-smoke.mjs Phase 8
        renames Counter→PropCounter and Widget→PropWidget to avoid the
        Phase 5 conflict. GC test (line ~803) updated to use PropCounter.
        Build passes; smoke runs past the original failure point.

Issue R2 (uncovered while validating R1): objc_interpreter.c:3987 had
        an orphan `return value_from_id((id)"(nil)");` inside the
        `if (cid > 0)` collection-dispatch block, BEFORE the handlers
        for valueForKey:, setValue:forKey:, and enumerateObjectsUsingBlock:.
        For any collection-typed receiver, that return fired before
        dispatch reached those handlers, making them unreachable.

        Symptom: smoke test `dict-kv` failed with
                 `[d setValue:@"hello" forKey:@"greeting"]; [d valueForKey:@"greeting"]`
                 returning `(nil)` instead of `hello`.
        Fix: 1-line deletion of the orphan return at line 3987. The
             closing `}` at line 3988 stays — it correctly closes
             `if (cid > 0)`, and valueForKey:/setValue:forKey:/enumerate
             handlers were already at the right scope (3756 anon-block
             level), they were just unreachable due to the early return.
        Status: FIXED in worktree, uncommitted.

Phase 1 — Structural refactor (12 incremental steps)
═══════════════════════════════════════════════════════════════════════════════

After Phase 0 lands, decompose objc_interpreter.c into ~13 cohesive modules.
This was the prior plan; reproduced and lightly amended here.

Decomposition (13 implementation files + 2 shared headers):

  kernel/
  ├── objc_interp_types.h         # ~180 lines — structs, enums, constants;
  │                                  string helpers as static inline;
  │                                  AstNode includes byte_start/byte_end
  │                                  fields (added now for Phase 3 reuse)
  ├── objc_interp_globals.h       # ~40 lines — extern declarations
  ├── objc_interp_lexer.{h,c}     # ~440 lines
  ├── objc_interp_value.{h,c}     # ~150 lines
  ├── objc_interp_string.{h,c}    # ~400 lines (string pool + GC)
  ├── objc_interp_state.{h,c}     # ~700 lines (var/method/property/coll
  │                                  /block/instance-var tables)
  ├── objc_interp_intern.{h,c}    # NEW ~150 lines (selector + identifier
  │                                  intern table; FNV-1a, open addressing)
  ├── objc_interp_ast.{h,c}       # ~200 lines (arena + constructors)
  ├── objc_interp_nslog.{h,c}     # ~500 lines (ring buffer + format)
  ├── objc_interp_foundation.{h,c} # ~1,200 lines (foundation_dispatch)
  ├── objc_interp_parser.{h,c}    # ~3,500 lines (parser + parse_message_send)
  ├── objc_interp_eval.{h,c}      # ~600 lines (AST eval, source range eval)
  ├── objc_interp_error.{h,c}     # NEW ~250 lines (frame stack, span→line/col,
  │                                  Rust-style render, did-you-mean)
  ├── objc_interp_inspect.{h,c}   # NEW ~200 lines (richer inspect bundle
  │                                  for objc_kernel_inspect_json)
  └── objc_interpreter.{h,c}      # ~200 lines (public API, global defs,
                                     objc_interp_reset_session)

Steps (each must produce a working `nix build .#kernel-wasm`):

   1. Create objc_interp_types.h and objc_interp_globals.h with declarations
      only. Build (no behavior change).
   2. Move cstr_copy, cstr_copy_checked, copy_identifier_or_error, cstr_eq,
      cstr_starts to types header as static inline. Build.
   3. Extract objc_interp_lexer.c (lines 224–662 + lexer-specific helpers).
   4. Extract objc_interp_value.c (Value factories + is_truthy).
   5. Extract objc_interp_nslog.c (NSLog + format_value + format_values_to_pool).
   6. Extract objc_interp_string.c (string_pool_alloc + objc_interp_gc_strings).
   7. Extract objc_interp_state.c (variable, collection, block,
      instance-var, method tables — including their lookup functions
      that Phase 2 will replace with hash-backed versions).
   8. Extract objc_interp_ast.c (arena + ast_make_*).
   9. Extract objc_interp_intern.c with stub (just an empty hash table
      for now; Phase 2 fills it in).
  10. Extract objc_interp_foundation.c. Refactor parse_message_send to
      call `int foundation_dispatch(class_name, sel_name, recv, args,
      arg_count, Value *result)` returning 0/1/-1 (not-handled / handled /
      handled-with-error). This is the seam that absorbed C3 fix.
  11. Extract objc_interp_eval.c (parse_statement_ast, parse_block_ast,
      eval_ast, eval_source_range).
  12. Extract objc_interp_parser.c. Update objc_interpreter.c to be
      ~200 lines: public API + global definitions + objc_interp_reset_session.
      Add objc_interp_error.c and objc_interp_inspect.c stubs.
      Update objc-jupyter-wasm/nix/kernel-wasm.nix to compile and link
      every new .c file.

Build flag changes during refactor (nix/kernel-wasm.nix):

  Add to clang flags:
    -Werror=implicit-function-declaration
    -Werror=incompatible-pointer-types
    -Wmissing-prototypes
  Temporarily DROP for the duration of the refactor:
    -Wno-unused-function
  This surfaces functions left orphaned by an extraction step — without
  it, dead static helpers slip through. Re-add at the end if real cruft
  remains.

After refactor is complete, every subsequent change is a small edit to
one or two modules, not surgery on a 9 KLOC file.

Phase 2 — Hash-backed lookups + selector interning (P1–P6)
═══════════════════════════════════════════════════════════════════════════════

Replace five linear scans on hot paths with open-addressing hash tables
keyed on cheap integer IDs. This is the single biggest perceptible speed
win.

  P1: coll_count                    — O(n) per [arr count]
  P2: coll_find_by_key              — O(n) per dict get / set membership
  P3: coll_get_nth                  — O(n) per [arr objectAtIndex:]
  P4: instance_var_get/set          — O(n) per synthesized property access
  P5: find_interpreter_method       — O(n) per message send
  P6: no selector interning         — strdup+strcmp every send

Implementation (in objc_interp_state.c and objc_interp_intern.c):

  Selector interning (objc_interp_intern.c):
    static char g_intern_pool[16 * 1024];      // backing storage
    static unsigned int g_intern_pool_used;
    static uint16_t g_intern_table[2048];      // index into g_intern_pool;
                                                  load factor ≤ 0.5
    static unsigned int g_intern_count;

    uint16_t intern(const char *s);            // FNV-1a, linear probe
    const char *intern_str(uint16_t id);

  Side index for collections (objc_interp_state.c):
    Keep g_coll_entries as the storage array. Add a parallel hash table
    keyed by (coll_id, intern_id_of_key) → index in g_coll_entries.
    Capacity 1024 entries (next pow2 over MAX_COLL_ENTRIES * 2).

  Method table key:  (class_ptr, intern(sel_name)) → method index
  Ivar table key:    (object_ptr, intern(prop_name)) → ivar index

Tunable: load factor 0.5 with linear probing. Apple's actual objc_msgSend
cache uses `(uintptr_t)sel & mask` with linear probing — same recipe.

Optional Phase 2b — monomorphic inline cache:
  Store last (class_id, method_idx) on each MessageSend AST node;
  fast-path skip the table lookup on repeat. ~20 LOC. Defer until
  profiling shows method dispatch dominates after Phase 2 lands.

Verification: micro-bench notebook in tests/run-notebooks.mjs that builds
a 400-element NSMutableArray, then loops `[arr objectAtIndex:i]` 1000×.
Snapshot wall-clock before/after via `console.time`. Target: ≥10× on
this micro-bench, and zero regression on existing demo notebooks.

Phase 3 — Error spans, frame stack, did-you-mean (D1, D2)
═══════════════════════════════════════════════════════════════════════════════

Lives in objc_interp_error.c. Surface every Phase 0 parser_error at the
exact source location, with a frame stack when the error fires inside a
user-defined method.

  AstNode spans:
    Add `uint32_t byte_start, byte_end` to AstNode struct.
    Lex once into a `static unsigned int g_line_starts[1024]` table.
    Resolve span → (line, col) by binary search on demand.

  Frame stack:
    static struct { uint16_t method_intern; uint16_t class_intern;
                    uint32_t span_start; } g_frames[64];
    static unsigned int g_frame_top;
    Push on execute_interpreter_method entry; pop on return.

  Render (Rust/Elm style) into g_error_buffer:
    error: <title>
       --> source.objc:<line>:<col>
        |
     L  | <code window>
        |     ^^^^^ <inline annotation>
        |
        = help: <suggestion or context>
    stack:
      <newest method>:<line> in <class>
      <next method>:<line> in <class>

  Did-you-mean:
    On unknown-selector or unknown-identifier, run Damerau–Levenshtein
    against the intern table; suggest closest match within edit distance 2.

Verification: regression notebook with a misspelled selector (`lenght`),
a misspelled class name, an out-of-range subscript (currently silently
returns "(nil)"; fix to error here too), and an exception in a method
called from another method. Snapshot expected output.

Phase 4 — Richer objc_kernel_inspect_json (D3)
═══════════════════════════════════════════════════════════════════════════════

The export already exists (nix/kernel-wasm.nix:80, --export=objc_kernel_inspect_json).
Its current body returns a near-empty bundle. Fill it in objc_interp_inspect.c.

Per Jupyter messaging spec: input is {code, cursor_pos, detail_level};
output is {status, found, data, metadata} where data is a MIME map.

Build for an identifier under cursor:

  Class:
    - Hierarchy chain (walk superclass via g_vars + cls→class_name lookup)
    - Declared methods (instance + class) with decoded type encodings
      (`v@:` → "(void)method:(id)arg")
    - Ivars: name, type encoding, declared owner (g_properties)
    - Conforming protocols (post Phase 5 if/when @protocol is parsed)

  Instance (when cursor is on a known variable):
    - Live ivar values via instance_var_get for every property of class
    - description / debugDescription if class implements them

  Selector:
    - Reverse-lookup: scan g_methods keyed by interned selector ID
      (cheap after Phase 2 lands), list classes that implement it

Render text/markdown (primary) and text/plain (fallback). Both go in
the data MIME bundle.

Verification: a kernel JS test harness invokes objc_kernel_inspect_json
on (a) NSMutableArray, (b) a user-defined class, (c) a known selector.
Assert MIME bundle includes the hierarchy / ivars / method signatures.
